Default navbar

Navbars are responsive meta components that serve as navigation headers for your application or site. They begin collapsed (and are toggleable) in mobile views and become horizontal as the available viewport width increases.

Overflowing content

Since Ark doesn't know how much space the content in your navbar needs, you might run into issues with content wrapping into a second row. To resolve this, you can:

  1. Reduce the amount or width of navbar items.
  2. Hide certain navbar items at certain screen sizes using responsive utility classes.

Requires Updating the Active Link

The Navbar requires the developer to update the current active link based on which navigation item is clicked. This is done by adding/removing an .active class to the li tags within the Navbar.

Responsive Support Requires JavaScript

In order for the Navbar to properly function when the menu items are collapsed, you will need to plug your own JavaScript code to do it, or you could use the Ark Navbar component in Ark Components.

If no JavaScript functionality is supplied, it will be impossible to expand the navbar and view the content within the .navbar-collapse
